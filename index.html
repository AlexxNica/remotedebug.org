RemoteDebug

Initiative to unify remote debugging across browsers.

The initiative.

RemoteDebug is an initiative to bring a common remote debugging protocol to today's browsers. It insn't a new phenomen. It has been around for years, but today each browser got their own, so we got:

  • Chrome Remote Debugging Protocol.
  • Opera Scope Transport Protocol (STP).
  • Mozilla Remote Debugging Protocol.
  • WebKit Remote Debugging Protocol (WIP).
  • Microsoft BrowserLink.

Each are incompatible, which means existing DevTools can't debug other browsers, and tooling implementor's need to spent a lot of time adding support for each protocol (and this means no-body does it).

The vision for this intiative is to enable in-browser DevTools and editors to communicate and remote multiple-browsers. Imagine if it was possible to use Chrome DevTools to debug Firefox, or use Firefox DevTools to debug Chrome. What if you could debug your app in IE and Chrome directly from Sublime? When running your beautiful PhoneGab-app inside an embeded WebView, you should be able to use your favorite debugging tool, and not be forced you to use Weinre.

We are building for the open web. Our tooling and protocols should be open too.

The goal.

The long-term goal is to get all major browser vendors to implement and use the RemoteDebug protocol, but we know this is going to take some time, especially to get everyone on board, so the first step is to provide proxy-implementations for all major browsers.

Once we get there, the next step is to get each browser vendor to implement RemoteDebug into their browser and their related DevTool.

The plan.

So how are we going to make this happen? Instead of starting from skretch, we propose to use the existing Chrome Remote debugging protocol as our starting point to define what we would call the "RemoteDebug protocol".

The Chrome protocol is a great starting point as it's open and [well-documented](link) protocol. The protocol is used by Chromium-based browsers, which means we already have Chrome and Opera supported. In addition we also have a Webkit-protocol-to Chrome proxy implemented, which could be useful, as more proxies are going to be added over time.

Get involved.

There's a lot of work ahead, so we need your help. You can help us by spreading the word or by contributing to the projects.

We need to make this happen. It's about time.
